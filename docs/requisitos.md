# Requisitos Funcionais (RF)  
*RF01*. O sistema deve permitir que o usuário realize cadastro e login por e-mail e senha.  
*RF02*. O sistema deve disponibilizar métricas de desempenho do usuario, como acertos, tempo de estudo e evolução.  
*RF03*. O sistema deve registrar todas as interções do usuário com conteúdos e atividades.  
*RF04*. O sistema deve permitir que administradores cadatrem, editem e removam conteúdos educacionais.  
*RF05*. O sistema deve recomendar conteúdos utilizando um modelo de IA.  
*RF06*. O sistema deve aplicar avaliações adaptativas, ajustando o nível de dificuldade das questões em tempo real.  
*RF07*. O sistema deve gerar trilhas de estudo personalizadas com basd no histórco de desempenho do usuário.
*RF08*. O sistema deve enviar notificações ao usu´rio sobre conteúdos, prazos e recomendações da IA.

# Requisitos Não Funcionais (RNF)  
*RNF01*. O sistema deve ser acessível via navegador web, sem necessidade de instalação local.  
*RNF02*. O tempo de carregamento das telas principais (Dashboard, Trilha, Aula) deve ser de até 2 segundos em condições normais.  
*RNF03*. O sistema deve garantir segurança de autenticação com criptografia de dados (HTTPS e hash de senha).  
*RNF04*. O sistema deve garantir controle de acesso, permitindo que cada usuário visualize apenas seus próprios dados.  
*RNF05*. A interface deve ser responsiva para uso em desktop e dispositivos móveis.  
*RNF06*. O sistema deve manter persistência e integridade dos dados de progresso do aluno.  
*RNF07*. O sistema deve registrar logs de ações relevantes (login, alteração de perfil, realização de avaliação, conclusão de exercícios).  
*RNF08*. O sistema deve garantir disponibilidade mínima de 99% mensal.
